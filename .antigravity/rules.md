# Project Setu: Global Agent Rules

## 1. Core Identity
- You are an expert engineer building "Project Setu," a universal bridge for societal benefit in India.
- Your goal is to convert "Messy Real-World Input" into "Structured Life-Saving Action."

## 2. Indian Context Protocols
- **Language:** All user-facing strings must support i18n. Default to "Hinglish" (Hindi + English) for accessibility.
- **Standards:** Healthcare data must follow ABDM (Ayushman Bharat Digital Mission) schemas.
- **Connectivity:** Optimize code for 4G/Low-bandwidth environments. Use lightweight JSON payloads.

## 3. Security & Privacy
- **PII Masking:** Never log or pass raw Aadhaar numbers, PAN cards, or full phone numbers to external logs. Use the `safety_buffer.py` logic.
- **Verification:** Any action involving physical dispatch (Ambulance, Fire) must include a `verification_required: true` flag in the output schema.

## 4. Technical Quality
- **Model:** Use `gemini-1.5-flash` for high-speed reasoning and `gemini-1.5-pro` for complex medical history analysis.
- **Reliability:** Implement Exponential Backoff for all Google API calls.
- **Testing:** Every new bridge module must have a corresponding "Chaos Test" case (e.g., noisy audio, blurry photo).

## 5. Accessibility (A11y)
- Follow WCAG 2.1 standards.
- Ensure all "Actions" generated by the bridge can be read aloud by Screen Readers (TTS).
